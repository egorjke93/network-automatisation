# Network Configuration Backup Script

Скрипт для автоматического снятия конфигураций с сетевых устройств различных производителей.

## Описание

Данный скрипт позволяет автоматизировать процесс резервного копирования конфигураций сетевого оборудования с поддержкой устройств:
- Cisco IOS
- Juniper JunOS
- Arista EOS
- Eltex и другие

## Функциональные возможности

- Подключение к устройствам по SSH с использованием Netmiko
- Автоматическое определение типа команды для снятия конфигурации
- Сохранение конфигураций в отдельную папку с датированием файлов
- Обработка enable-режима (при необходимости)
- Логирование процесса работы
- Поддержка таймаутов для стабильной работы с медленными устройствами

Настройка
1. Создание файла devices_ips.py
Перед использованием необходимо создать файл devices_ips.py со списком устройств:

python
devices_list = [
    {
        'device_type': 'cisco_ios',
        'host': '192.168.1.1',
        # Дополнительные параметры при необходимости
    },
    {
        'device_type': 'juniper',
        'host': '192.168.1.2',
    },
    {
        'device_type': 'arista_eos', 
        'host': '192.168.1.3',
    },
    # Добавьте свои устройства
]
2. Поддерживаемые device_type
cisco_ios - для оборудования Cisco IOS

juniper - для оборудования Juniper JunOS

arista_eos - для оборудования Arista EOS

Другие типы, поддерживаемые Netmiko

Использование
Запустите скрипт командой:

bash
python backup_script.py
Скрипт запросит:

Имя пользователя для подключения к устройствам

Пароль для подключения

Enable пароль (если требуется, иначе нажмите Enter)

Результат работы
Конфигурации сохраняются в папку config_backups/

Имена файлов формируются по шаблону: {hostname}_backup_{дата}.txt

Логи работы сохраняются в файл netmiko_global.log

Особенности работы с разными устройствами
Cisco IOS/Arista
Используется команда show running-config

Juniper
Используется команда show configuration | display set

Особенности для Eltex
Для устройств Eltex реализована специальная обработка с использованием:

Явного указания expect_string для ожидания промпта

Увеличенных таймаутов чтения (read_timeout=120)

Логирование
Скрипт ведет подробное логирование в файл netmiko_global.log, что помогает в диагностике проблем подключения.

Безопасность
Пароли вводятся интерактивно через getpass и не сохраняются в файлах

Рекомендуется ограничить права доступа к файлам с конфигурациями

Возможные проблемы и решения
Таймаут подключения - увеличьте значение conn_timeout в devices_list

Таймаут выполнения команд - увеличьте read_timeout в функции backup_config

Проблемы с промптом - проверьте корректность device_type и при необходимости настройте expect_string
